### Local voice dictation on Linux

Surveyed fully local (no cloud, no API key) voice dictation options for NixOS + Wayland. Whisper weights are open-source (MIT license); downloaded once from HuggingFace (model registry).

| Engine | Tool | Wayland | Punctuation | Latency (CPU) | Notes |
|---|---|---|---|---|---|
| Whisper | `whisper.cpp` | Yes (`wtype`) | Yes | ~400ms (`base.en`) | ‚úÖ Recommended |
| Whisper | `faster-whisper` | Yes (`wtype`) | Yes | ~300ms | Python, CTranslate2 (optimised inference) backend |
| Whisper | `whisper-live` | Yes | Yes | ~300ms streaming | Harder setup |
| Vosk | `nerd-dictation` | Partial (XWayland only) | No | ~100ms | Breaks on native Wayland apps |

!!! tip "All options are fully local ‚Äî no API key, no cloud"
    `wtype` = Wayland-native keystroke injector (replaces `xdotool` which only works in XWayland compatibility mode).

**Whisper model sizes** (all local, pick by CPU speed):

| Model | Size | CPU latency | Notes |
|---|---|---|---|
| `tiny.en` | 75 MB | ~150ms | Fast, rough |

### Linux audio landscape

Three layers stack on top of each other. Knowing which layer a tool lives at explains why `pactl` doesn't exist here.

| Layer | What it is | Servers / implementations | CLI tools |
|---|---|---|---|
| **Apps** | Your programs | sox, ffmpeg, browsers, etc. | `rec` `play` `pw-play` `aplay` |
| **Sound server** (userspace) | Routes audio between apps and hardware | **PipeWire** ‚Üê this machine ¬∑ PulseAudio (older, Ubuntu etc) ¬∑ JACK (pro/low-latency audio) | `wpctl` (PipeWire) ¬∑ `pactl` (PulseAudio) ¬∑ `jack_*` |
| **ALSA** (kernel) | Raw hardware drivers | Built into the Linux kernel | `aplay` `arecord` `alsamixer` |

**PipeWire** (2021+) replaces both PulseAudio and JACK (Just Another Computer Kitchen) with a single modern server. It can optionally expose compatibility shims so old tools still work:

- `pipewire-pulse` shim ‚Üí makes `pactl`/`paplay`/`parecord` available
- `pipewire-jack` shim ‚Üí makes the JACK API available

!!! warning "Why `pactl` is missing on this machine"
    Pure PipeWire with no `pipewire-pulse` shim installed. `wpctl` is the native replacement.
    To restore `pactl`: add `pipewire-pulse` to NixOS config.

| Task | PulseAudio cmd | PipeWire-native equivalent |
|---|---|---|
| List devices | `pactl list sources short` | `wpctl status` |
| Set volume | `pactl set-sink-volume @DEFAULT_SINK@ 80%` | `wpctl set-volume @DEFAULT_AUDIO_SINK@ 80%` |
| Inspect device | `pactl list cards` | `wpctl inspect <id>` |
| Record | `parecord file.wav` | `pw-record file.wav` |
| Play | `paplay file.wav` | `pw-play file.wav` |

### Bluetooth audio profiles & the mic quality degradation problem

When you open a mic input, the headset auto-switches from A2DP ‚Üí HFP ‚Äî audio quality tanks. This is a Bluetooth hardware constraint, not a software bug.

| Profile | Full name | Quality | Direction | Codec | Used when |
|---|---|---|---|---|---|
| **A2DP** | Advanced Audio Distribution Profile | High (44.1kHz stereo) | One-way (listen only) | LDAC / aptX / AAC | Music, video |
| **HSP** | Headset Profile | Poor (8kHz mono) | Bidirectional | CVSD | Basic calls |
| **HFP** | Hands-Free Profile | OK (16kHz mono) | Bidirectional | CVSD / mSBC (wideband) | Calls, mic use |

Root cause: Bluetooth uses a SCO (Synchronous Connection Oriented) link for bidirectional audio. This narrow channel can't carry A2DP (Advanced Audio Distribution Profile)-quality data in both directions simultaneously ‚Äî the bandwidth simply isn't there.

!!! tip "HFP is fine for Whisper"
    HFP at 16kHz is exactly Whisper's training format ‚Äî quality is sufficient. The degradation only matters if you want music playing *at the same time* as dictating.

!!! success "Fix: use a separate mic source"
    Point `rec`/`pw-record` at the built-in mic or a USB mic. Headphones stay in A2DP (high quality) because no app is using the headset mic channel.

#### How to see which profile is active

```bash
wpctl inspect <sink-node-id>   # get id from: wpctl status
# look for:
#   api.bluez5.profile = "a2dp-sink"   ‚Üê high quality
#   api.bluez5.profile = "headset-head-unit"  ‚Üê HFP/HSP degraded
#   api.bluez5.codec   = "ldac"        ‚Üê codec in use
```

Current state of WH-1000XM4 on this machine (confirmed 2026-06-06):
- Profile: `a2dp-sink` (high quality)
- Codec: `ldac`
- Built-in mic (`Ryzen HD Audio Controller Stereo Microphone`) is the default source ‚Äî headphones are NOT the mic source, so A2DP is preserved
